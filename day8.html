  • Gitについて

    1. コミット前の変更ファイル欄に表示される横棒入りのファイルについて
      2. ファイル名を変更したとき、stage待ちのファイル群の中には、同じファイルであって名前が異なる二つのもの（古い名前のものと新しい名前のものが存在している。 そして、その古い名前がつけられた方のファイルには横線が入る。

    2. プルリクエストの"file changes"について
      3. プルリクエストの"file change"欄に出てくるファイルたちは、実はリモートのmasterブランチ（最新のmasterということ）に対するfeatureブランチとの差分を見ている。 てっきりfeatureブランチが切られたときのmasterとの差分だと思ってたから意外だった。

    3. conflictの解消の仕方
      4. 引き継ぎ課題の場合は、多分ローカルのfeatureにリモートのmasterをpullすればいいんだと思う。つまり、「featureブランチにいる状態で」（超大事）

      git pull origin master

      とすればいいと思う。そうすれば当然conflict起きてるんだからpullできませんでしたみたいなメッセージとともに例の画面が出てくるはず。

    4. git push origin "featureブランチ"というコマンドについて
      5. このコマンド、実は指定したfeatureブランチがリモートに存在していない場合、そのブランチをリモートに作ってくれるということが判明した。 つまり、新規課題始めるときローカルでfeature切って作業してコミットまでしたら、（リモートにそのfeatureはまだないのに）

      git push origin "featureブランチ"

      とだけ打てばよいということ。
