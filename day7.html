  • Gitについて
    • 環境構築の段階でpvs2をクローンしたとき、実は"v2/master"という感じでmasterブランチのみを指定していたことが分かった。 よって、引き継ぎ課題に着手する際、既に誰かによって作成されたfeatureブランチをローカルに持ってこなきゃいけなくて、 そのためのコマンドは（おそらく）

    git switch -c ブランチ名 origin/ブランチ名

    でよいと思う（明日要確認）。また、リモート含めたすべてのブランチを見るには

    git branch -a

    と打てばよいことが分かった。
